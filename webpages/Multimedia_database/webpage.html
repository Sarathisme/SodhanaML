Multimedia database

\t\t\t
\n\t\t\t\t
From Wikipedia, the free encyclopedia
\t\t\t\t
\n\t\t\t\t
\t\t\t\tJump to navigation\n\t\t\t\tJump to search\n\t\t\t\t

A Multimedia database (MMDB) is a collection of related multimedia data.[1] The multimedia data include one or more primary media data types such as text, images, graphic objects (including drawings, sketches and illustrations) animation sequences, audio and video.\n

A Multimedia Database Management System (MMDBMS) is a framework that manages different types of data potentially represented in a wide diversity of formats on a wide array of media sources. It provides support for multimedia data types, and facilitate for creation, storage, access, query and control of a multimedia database.[2]\n

Contents of MMDB[edit]

\n

A Multimedia Database (MMDB) hosts one or more multimedia data types[3] (i.e. text, images, graphic objects, audio, video, animation sequences). \nThese data types are broadly categorized into three classes: \n

\n
  • Static media[3] (time-independent: image and graphic object).
    • \n
  • Dynamic media[3] (time-dependent: audio, video and animation).
    • \n
  • Dimensional media[3](3D game and computer aided drafting programs).
\n

Comparison of multimedia data types[edit]

\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n
MediumElementsTime-dependence\n
TextPrintable charactersNo\n
GraphicVectors, regionsNo\n
ImagePixelsNo\n
AudioSound, VolumeYes\n
VideoRaster images, graphicsYes\n
\n

Additionally, a Multimedia Database (MMDB) needs to manage additional information pertaining to the actual multimedia data. \nThe information is about the following: \n

\n
  • Media data: the actual data representing an object.
    • \n
  • Media format data: information about the format of the media data after it goes through the acquisition, processing, and encoding phases.
    • \n
  • Media keyword data: the keyword descriptions, usually relating to the generation of the media data.
    • \n
  • Media feature data: content dependent data such as contain information about the distribution of colours, the kinds of textures and the different shapes present in an image.
\n

The last three types are called metadata as they describe several different aspects of the media data. The media keyword data and media feature data are used as indices for searching purpose. The media format data is used to present the retrieved information.\n

\n

Requirements of Multimedia databases[edit]

\n

Like the traditional databases, Multimedia databases should address the following requirements:\n

\n
  • Integration\n
    • Data items do not need to be duplicated for different programs invocations
    • \n
  • Data independence\n
    • Separate the database and the management from the application programs
    • \n
  • Concurrency control\n
    • Allows concurrent transactions
    • \n
  • Persistence\n
    • Data objects can be saved and re-used by different transactions and program invocations
    • \n
  • Privacy\n
    • Access and authorization control
    • \n
  • Integrity control\n
    • Ensures database consistency between transactions
    • \n
  • Recovery\n
    • Failures of transactions should not affect the persistent data storage
    • \n
  • Query support\n
    • Allows easy querying of multimedia data
\n

Multimedia databases should have the ability to uniformly query data (media data, textual data) represented in different formats and have the ability to simultaneously query different media sources and conduct classical database operations across them. (Query support)\n

They should have the ability to retrieve media objects from a local storage device in a good manner. (Storage support)\n

They should have the ability to take the response generated by a query and develop a presentation of that response in terms of audio-visual media and have the ability to deliver this presentation. (Presentation and delivery support)\n

\n

Issues and challenges[edit]

\n
  • Multimedia data consists of a variety of media formats or file representations including TIFF, BMP, PPT, IVUE, FPX, JPEG, MPEG, AVI, MID, WAV, DOC, GIF, EPS, PNG, etc. Because of restrictions on the conversion from one format to the other, the use of the data in a specific format has been limited as well. Usually, the data size of multimedia is large such as video; therefore, multimedia data often require a large storage.
    • \n
  • Multimedia database consume a lot of processing time, as well as bandwidth.
    • \n
  • Some multimedia data types such as video, audio, and animation sequences have temporal requirements that have implications on their storage, manipulation and presentation, but images, video and graphics data have special constraints in terms of their content.
